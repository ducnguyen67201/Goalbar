# Live test runbook

Live tests are manual and opt-in.

1. Use dedicated non-production platform accounts.
2. Confirm `docs/platform-access.md` is current.
3. Set platform spend limits where available.
4. Connect through the official consent screen.
5. Publish a clearly labeled disposable test post only after confirming its preview.
6. Test only capabilities granted to the application.
7. Revoke the token and disconnect after the run.
8. Review local logs for the secret sentinel before sharing them.

General LinkedIn DMs are not a live-test step without separately approved private access.
